# Screenshot Capture Instructions

Capture screenshots manually from the current running application. Save each image under `docs/images/` using the exact filename below so all Markdown placeholders render without link changes.

| Filename | What to capture |
| --- | --- |
| `login-staff-mobile.png` | Mobile-width `/login` page showing the Staff demo card. |
| `staff-pickup-request.png` | Completed pickup form with cargo type, generated zone, and location grid. |
| `staff-missions.png` | Mission list showing lifecycle actions and processed decision details. |
| `staff-live-map-show-all.png` | Live Map in Show All Robots mode. |
| `staff-live-map-selected-robot.png` | Selected robot view with mission flow, battery, and strategy. |
| `manager-policy-assignment.png` | Zone policy cards and active Admin rules. |
| `manager-robot-task-board.png` | Robot workload groups, charging state, and task details. |
| `admin-rule-management.png` | Rule form and rule list. |
| `admin-robot-management.png` | Fleet cards with backend-derived status. |
| `execution-simulator.png` | Simulator result with rule and condition traces. |
| `system-flow.png` | Architecture flow and pattern sections. |

## Capture checklist

1. Use representative demo data; do not expose private production data.
2. Keep browser zoom consistent across related screenshots.
3. Capture Staff pages at a phone-friendly width where requested.
4. Ensure status and strategy labels are readable.
5. Do not crop out the page title or the feature being documented.
6. Re-capture an image when UI behavior changes; do not edit screenshots to imply unsupported features.

No automated screenshot generation is required for this documentation package.
